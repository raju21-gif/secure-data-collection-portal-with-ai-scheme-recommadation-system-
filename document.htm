1. SYNOPSIS

The "AI-Powered Scheme & Job Recommendation Portal" is a revolutionary digital intervention designed to democratize access to government welfare and employment opportunities. Traditional systems for scheme discovery are often fragmented, requiring high digital literacy and complex navigation. This project addresses these challenges by introducing a multimodal interface featuring voice-to-text form filling and a hybrid AI recommendation engine.

The system leverages state-of-the-art Natural Language Processing (NLP) using Sentence Transformers to match user semantic profiles with detailed scheme descriptions. Additionally, a Backpropagation Neural Network is utilized for quantitative scoring based on eligibility criteria such as age, income, and occupation. The frontend is developed using React.js 18, providing a seamless user experience, while the backend is powered by FastAPI and MongoDB for secure and scalable data management. This project represents a significant leap toward inclusive digital governance.

2. INTRODUCTION

2.1 Background

In a rapidly digitizing economy, government schemes for agriculture, healthcare, and education play a vital role. However, the information gap often prevents the most vulnerable populations from accessing these benefits. This project aims to bridge this gap using Artificial Intelligence.

2.2 Problem Statement

Existing portals require manual searching and complex filtering. Users often find it difficult to identify schemes they are eligible for among thousands of options. Furthermore, typing long forms is a barrier for semi-literate users.

2.3 Objectives

  • To develop an AI-driven platform for personalized scheme and job recommendations.
  • To integrate voice-enabled data collection to enhance accessibility.
  • To provide a secure administrative dashboard for real-time monitoring of user engagement.
  • To implement a hybrid scoring model for high-precision resource matching.

3. SYSTEM ANALYSIS

3.1 PROPOSED SYSTEM

The proposed system introduces an automated, voice-interactive web portal. Unlike static websites, this system "reasons" about user data. It features:

  • AI Chatbot (Keran): A conversational interface to guide users.
  • Context-Aware Matching: Semantic search using 384-dimensional sentence embeddings.
  • Automated Eligibility Validation: Real-time checks against user profile data.

3.2 EXISTING SYSTEM

The current manual or legacy digital systems depend on text-heavy directories. Limitations include:

  • Lack of personalization.
  • Manual filtering required by the user.
  • Barriers for visually impaired or digitally unskilled users.
  • Disconnected job and welfare information.

3.3 SOFTWARE REQUIREMENT SPECIFICATION (SRS)

3.3.1 Functional Requirements

  • User Login and Registration with profile image upload.
  • Voice-based input for multi-field forms.
  • Automated recommendation algorithm based on profile analysis.
  • Admin tracking of viewed and applied schemes.

3.3.2 Non-Functional Requirements

  • Reliability: System must maintain 99.9% uptime for critical welfare access.
  • Security: All user data protected via JWT and encrypted hashing.
  • Performance: AI matching result generation within < 2 seconds.

3.3.3 Hardware Requirements

  • Processor: Intel i5 or equivalent (Minimum).
  • RAM: 8GB (Recommended 16GB for AI model hosting).
  • Storage: 20GB Free space for MongoDB and Dataset storage.

3.3.4 Software Requirements

  • Operating System: Windows 10/11 or Linux.
  • Frontend: React.js, Tailwind CSS.
  • Backend: Python 3.12, FastAPI.
  • Database: MongoDB 6.0+.

3.4 FEASIBILITY ANALYSIS

Technology Feasibility: The use of Python and React is highly feasible due to community support and existing libraries for AI and Web Speech.

Economic Feasibility: Open-source tools minimize costs, making it sustainable for large-scale deployment.

4. SYSTEM DESIGN

4.1 Architecture Design

The system follows a Modern MERN-like stack where Express is replaced by FastAPI for superior Python/AI integration. The frontend communicates with the backend via RESTful APIs.

4.2 Database Schema

Collection: Users

  • name: String
  • email: String (Unique)
  • password: Hashed (Bcrypt)
  • role: String (Admin/User)
  • image_url: String (Cloud/Local path)

5. SYSTEM TESTING AND IMPLEMENTATION

5.1 Unit Testing

Each module, including the Voice Capture Hook and the Recommendation Engine, underwent rigorous unit testing to ensure data integrity during transcription and scoring.

5.2 Integration Testing

Ensured that the frontend React pages correctly consume the FastAPI recommendation endpoints and that the JWT token is persisted across sessions.

6. SECURITY TECHNOLOGIES AND POLICIES

The application implements enterprise-level security protocols:

  • JWT Authentication: Secure token-based access for private routes.
  • Password Hashing: SHA-256 with Bcrypt salting to prevent rainbow table attacks.
  • Role-Based Access Control (RBAC): Restricting admin analytics to authorized personnel only.

7. MAINTENANCE

Post-deployment maintenance involves periodic updates of the scheme and job datasets (CSV files), monitoring API latency, and retraining AI models as new eligibility patterns emerge.

8. CONCLUSION

The "AI-Powered Scheme & Job Recommendation Portal" successfully demonstrates the application of modern web technologies and AI in solving critical social governance problems. By simplifying the discovery process, it empowers users to take advantage of opportunities effectively.

9. SCOPE FOR FUTURE ENHANCEMENT

  • Multilingual Support: Extending voice recognition to regional Indian languages.
  • Blockchain Integration: For immutable storage of applied scheme records.
  • Direct Application: API integration with government portals for 1-click application.
